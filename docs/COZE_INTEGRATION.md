# COZE_INTEGRATION

## Standard Call Order

Recommended step-by-step Coze workflow:

1. `POST /coze/project/init`
2. `POST /characters/{character_id}/confirm-reference`
3. `POST /coze/project/{project_id}/generate-script`
4. `POST /coze/project/{project_id}/storyboard`
5. `POST /coze/project/{project_id}/create-asset-tasks`
6. `POST /coze/project/{project_id}/run-asset-tasks`
7. `GET /coze/project/{project_id}/summary`
8. `POST /coze/project/{project_id}/publish-record`
9. `GET /coze/project/{project_id}/summary`

## Simplified Demo Call

For easier Coze debugging, use:

- `POST /coze/project/full-demo-flow`

This is a demo-only orchestration endpoint that wraps the full MVP flow in one call.

## Payload Sources

- `project_card_json`: project metadata generated by Coze
- `characters_json`: role definitions generated by Coze
- `script_card_json`: structured script card generated by Coze
- `storyboard_json`: structured storyboard generated by Coze
- `video_shot_ids`: storyboard shot ids that also need video generation
- `publish_record_json`: publish metadata for the demo closure

## next_action Guide

- `confirm_character_reference`
  Project exists, but no confirmed character reference yet.
- `create_storyboard`
  Script or project exists, but no shots exist yet.
- `create_asset_tasks`
  Shots exist, but no asset tasks exist yet.
- `run_asset_tasks`
  Asset tasks exist, but some queued or retryable tasks still need execution.
- `review_failed_tasks`
  Some tasks failed or need human revision.
- `ready_to_publish`
  All asset tasks succeeded.
- `completed`
  Publish record exists and project status is `published`.

## Public URL Reminder

Coze cannot directly access `localhost` or `127.0.0.1`.  
Use one of these for external access:

- `ngrok`
- `Cloudflare Tunnel`
- a deployed public service URL

For local integration testing, expose your FastAPI service first and then configure the public URL in Coze.
